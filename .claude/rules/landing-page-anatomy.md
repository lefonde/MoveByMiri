# Landing Page Anatomy

> **MOOV by Miri**: This is the generic framework. For MOOV-specific pages, see `rules/moov-station-map.md` which maps these 8 sections to the brand's 10-station structure.

Standard section structure for landing and marketing pages. Use this as a starting framework — sections can be reordered or omitted with justification, but the default order is optimized for conversion.

## 1. Promise / Hero

- Single clear value proposition headline (benefit-oriented, not feature-oriented)
- Supporting subheadline (one sentence expanding on the promise)
- Primary CTA above the fold
- Optional supporting visual (product screenshot, illustration, or video)
- No navigation clutter — minimal header

## 2. Social Proof

- Positioned immediately after the promise to build trust
- Must be specific: real numbers, real names, recognizable logos
- Formats: logo bar, testimonial quotes, metric highlights
- Never vague ("trusted by thousands") — always concrete ("used by 2,400 teams")

## 3. Problem Framing

- Articulate the pain point the audience recognizes
- Use the audience's language, not internal jargon
- Create tension that the product resolves
- Keep it brief — 2-3 sentences or a comparison

## 4. Feature Architecture

- 3-5 key features maximum (not a feature dump)
- Each feature: benefit-first headline + supporting detail + optional visual
- Visual hierarchy makes scanning easy (icon + headline + one line)
- Features should map to the problems framed above

## 5. Product Demo

- Screenshot, video, or interactive element showing the product in action
- Annotated if needed — don't assume the visual is self-explanatory
- Shows a realistic use case, not an empty state
- Optional: embedded interactive demo or animated walkthrough

## 6. Objection Handling

- FAQ, comparison table, or "how it works" section
- Addresses the top 3-5 hesitations (price, complexity, migration, trust, support)
- Tone: confident and direct, not defensive

## 7. CTA Repetition

- Repeat the primary CTA with a different angle or increased urgency
- Can include a secondary CTA (e.g., "Talk to sales" alongside "Start free trial")
- Placement after the user has seen features, proof, and objection handling

## 8. Footer Trust Layer

- Legal links (privacy, terms)
- Secondary navigation (about, blog, docs, careers)
- Contact information or support link
- Trust badges (security certifications, partner logos)
- Copyright notice
